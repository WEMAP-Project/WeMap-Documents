    Map matching API

    Map matching API giúp bạn có thể biến tập tin GPX (lưu trữ nhật ký GPS) thành thông tin tuyến đường chính xác để phục vụ một số công việc như: làm sạch dữ liệu, thêm thông tin cho dữ liệu như độ cao, thêm chỉ dẫn cho tuyến đường,...

    Thông tin về chi phí sử dụng dịch vụ vui lòng xem tại Thông tin bảng giá dịch vụ.

    POST "https://routing.geocode.vn/match?key={API_KEY_CỦA_BẠN}
    Thông số bắt buộc Loại dữ liệu Miêu tả
    key Chuỗi API key của bạn. Cách khởi tạo một API key xem tại đây.

Các tuỳ chọn khác: bike (xe đạp), foot(đi bộ), hike (leo núi), truck (xe tái),small_truck (xe tái nhỏ) | locale | Chuỗi | Ngôn ngữ hướng dẫn chỉ đường. Mặc định vi (tiếng Việt). Chọn true nếu bạn muốn thêm tham số độ cao số vào dẫn đường. | details | Mảng các chuỗi | Các thông tin chi tiết bạn muốn biết thêm về tuyến đường. Các thông tin bao gồm: street_name (tên đường), max_speed (vận tốc tối đa), toll (phí), lanes (số làn), surface (loại bề mặt đường) | optimize | Bu lì ừn | Sử dụng khi bạn muốn tối ưu khi được dẫn đường qua nhiều hơn 2 điểm. Mặc định là false. Chọn true nếu bạn không cần qua các điểm ở giữa lần lượt, chúng tôi sẽ tìm đường đi được tối ưu cho bạn khi phải qua tất cả các điểm đã chọn. | instructions | Bu lì ừn | Các chỉ dẫn việc di chuyển trên tuyến đường tìm được. Mặc định là true. Chọn false nếu bạn muốn ẩn các chỉ dẫn này. | type | Chuỗi | Loại dữ liệu bạn muốn trả về. Mặc định là json. Các lựa chọn khác: gpx.

    Ví dụ Matching file GPX bên trên

    curl -X POST \
  'https://routing.wemap.vn/match?locale=v&gps_accuracy=20&max_visited_nodes=3000&type=json&instructions=true&details=street_name&elevation=false&vehicle=car&key={API_KEY_CỦA_BẠN}' \
  -H 'Accept: */*' \
  -H 'Accept-Encoding: gzip, deflate' \
  -H 'Cache-Control: no-cache' \
  -H 'Connection: keep-alive' \
  -H 'Content-Length: 1106' \
  -H 'Content-Type: application/gpx+xml' \
  -H 'Host: routing.wemap.vn' \
  -H 'cache-control: no-cache' \
  -d '<?xml version="1.0" encoding="UTF-8"?>
<gpx>
    <trk>
        <trkseg>
            <trkpt lat="21.03664023419462" lon="105.78187885461377"><time>2019-01-01T00:00:00Z</time></trkpt>
            <trkpt lat="21.036645241079803" lon="105.78203978715466"><time>2019-01-01T00:01:00Z</time></trkpt>
            <trkpt lat="21.036645241079803" lon="105.7821578043513"><time>2019-01-01T00:02:00Z</time></trkpt>
            <trkpt lat="21.036645241079803" lon="105.78224363503979"><time>2019-01-01T00:03:00Z</time></trkpt>
            <trkpt lat="21.036650247964822" lon="105.78234555898234"><time>2019-01-01T00:04:00Z</time></trkpt>
            <trkpt lat="21.03664023419462" lon="105.7824260252528"><time>2019-01-01T00:05:00Z</time></trkpt>
            <trkpt lat="21.036625213538066" lon="105.78250649152325"><time>2019-01-01T00:06:00Z</time></trkpt>
            <trkpt lat="21.036620206652223" lon="105.78260305104779"><time>2019-01-01T00:07:00Z</time></trkpt>

    Các mã lỗi của API Tìm kiếm

    Trường message trong phần thân phản hồi Mã lỗi HTTP Miêu tả
    Yêu cầu của bạn không đúng 400 Khi bạn gửi lên yêu cầu quá ít hoặc quá nhiều points
    Lỗi xác thực - Thiếu API key 401 Thông số key là bắt buộc
    Lỗi xác thực - Sai API key 401 Kiểm tra lại thông số key
    Lỗi hệ thống 500 Chúng tôi đã nhận được thông tin và sẽ khắc phục trong thời gian sớm nhất.
    Lỗi lựa chọn phương tiện 501 Kiểm tra lại thông số vehicle. Hệ thống chỉ hỗ trợ các loại phương tiện đã liệt kê
